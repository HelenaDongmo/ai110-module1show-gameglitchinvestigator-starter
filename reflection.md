## 4. What did you learn about Streamlit and state?

- In your own words, explain why the secret number kept changing in the original app. The secret number kept changing because Streamlit reruns the entire script every time the user interacts with the app
- How would you explain Streamlit "reruns" and session state to a friend who has never used Streamlit? Imagine Streamlit as a magic script that restarts from the beginning every time you do anything in the app—like clicking a button or typing in a box, it's not like a normal website that remembers things on the server. Reruns mean the whole code runs again, which is why things can reset if you're not careful. Session state is like a special backpack that carries your important stuff (like scores or secrets) from one rerun to the next, so your app can remember what's happened without losing progress.
- What change did you make that finally gave the game a stable secret number?
I added checks using st.session_state to store the secret number persistently across reruns.
